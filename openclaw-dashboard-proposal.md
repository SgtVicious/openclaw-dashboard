## Executive Summary

## Page Types & Views

### 1. Dashboard (Monitoring Page)
- **Purpose**: Real-time overview of all OpenClaw activity
- **Key Elements**:
  - KPI cards (active agents, sessions, performance)
  - Live activity charts
  - Resource monitor (Pi CPU/Memory/Temp)
  - Recent sessions table
  - Live log stream
  - Quick action buttons

### 2. Agents Management Page
- **Purpose**: CRUD operations for agents and sub-agents
- **Key Elements**:
  - Kanban board view (Scheduled â†’ Running â†’ Complete â†’ Reviewed)
  - List view with filtering/sorting
  - Agent detail view with:
    - Configuration editor
    - Session history
    - Performance metrics
    - Sub-agent tree visualization
  - Bulk operations (start/stop/delete)

### 3. Configuration Editor Page
- **Purpose**: Edit JSON configs and Markdown files via UI
- **Key Elements**:
  - File browser sidebar
  - JSON editor with schema validation
  - Markdown editor with live preview
  - Syntax highlighting
  - Auto-save with version history
  - Import/Export functionality

### 4. Logs & Monitoring Page
- **Purpose**: View and search all OpenClaw logs
- **Key Elements**:
  - Filterable log viewer (by agent, level, time)
  - Search with regex support
  - Export to file
  - Real-time tail mode
  - Log level color coding

### 5. System Status Page
- **Purpose**: Monitor Pi health and OpenClaw daemon
- **Key Elements**:
  - System resources (CPU, Memory, Disk, Temp)
  - OpenClaw service status
  - Gateway connection status
  - LLM model status
  - Restart/Stop controls

## Data Models

### Agent Model
```typescript
interface Agent {
  id: string;
  name: string;
  description: string;
  status: 'idle' | 'running' | 'error' | 'stopped';
  configPath: string;      // path to agent's .md file
  jsonConfigPath: string;  // path to agent's .json config
  parentAgentId?: string;  // for sub-agents
  subAgentIds: string[];
  createdAt: Date;
  lastActiveAt: Date;
  sessionCount: number;
  totalTasksCompleted: number;
  averageResponseTime: number;
}
```

### Session Model
```typescript
interface Session {
  id: string;
  agentId: string;
  agentName: string;
  status: 'active' | 'completed' | 'error';
  startTime: Date;
  endTime?: Date;
  tasks: Task[];
  logs: LogEntry[];
  sessionDataPath: string;
}
```

### Configuration Model
```typescript
interface ConfigFile {
  path: string;
  name: string;
  type: 'json' | 'markdown';
  content: string;
  lastModified: Date;
  size: number;
  schema?: object;  // for JSON validation
}
```

## SSH Access Setup (LAN)

### From ROG Laptop to Pi:
```bash
# One-time setup: Add Pi to known hosts
ssh pi@raspberrypi.local

# Create SSH tunnel for dashboard access
# This forwards port 3000 on your laptop to port 3000 on the Pi
ssh -L 3000:localhost:3000 pi@raspberrypi.local

# Now access dashboard at: http://localhost:3000 on your ROG laptop
```

### Alternative: Autossh for persistent tunnel
```bash
# Install autossh for automatic reconnection
sudo apt install autossh

# Create persistent tunnel
autossh -M 0 -N -L 3000:localhost:3000 pi@raspberrypi.local
```
